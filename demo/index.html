CoCreate Plugins API

A declarative, attribute-based system for initializing and configuring JavaScript libraries.

1. The Basics: Declarative Initialization

Initialize any supported plugin using the plugin="PluginName" attribute. Configure it using the lowercase attribute matching the plugin name (e.g., pluginname="{ options }").

Live Demo
Code
<button 
    plugin="Toastify" 
    toastify='{ 
        "text": "Hello!", 
        "duration": 3000, 
        "style": { "background": "#198754" } 
    }'
    onclick="this.Toastify.showToast()">
    Click Me
</button>

2. Variable Operators

The API supports special variables to reference DOM elements and global objects dynamically within your JSON configuration.

Operator Description Usage Example
$this References the current DOM element where the attribute is placed. "element": "$this"
$window Accesses the global window object (useful for callbacks). "callback": "$window.console.log"
Example: Using $this for Element Binding
<div 
    plugin="raterJs" 
    raterjs='[{ 
        "element": "$this", 
        "starSize": 32, 
        "rating": 3.5 
    }]'>
</div>

3. Configuration Patterns (Array vs. Object)

Plugins have different constructor signatures. The API handles both by inspecting the JSON structure.

Pattern A: Single Config Object

Used when the plugin accepts a single object containing all settings (e.g., raterJs({ element: div })).

config='[{ "element": "$this", "opt": "val" }]'
*Note: Enclosed in an array to represent arguments list.
Pattern B: Argument List

Used when the plugin accepts multiple distinct arguments (e.g., Swiper(element, options)).

config='["$this", { "opt": "val" }]'

4. Advanced: Callbacks & Global Functions

Use $window or specific global names (like $Swal) to pass functions into configuration attributes.

<button 
    onclick="Swal.fire({
        title: 'Ready?',
        preConfirm: '$window.myCustomFunction' 
    }) >
    Launch
</button>
